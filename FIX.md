# Разбор инцидента с принтером HP

## Initial situation

В корпоративной сети был зафиксирован инцидент, связанный с сетевым принтером HP. Пользователи сообщили о бесконечной печати листов с некорректным содержимым. В журнале устройства наблюдалось большое количество заданий от пользователя Guest, что потребовало проведения дополнительного расследования.

## Task log analysis

Для начала был выполнен просмотр журнала печати (Job Log) через веб-интерфейс принтера.
<img width="1178" height="300" alt="image" src="https://github.com/user-attachments/assets/69ec1ba6-edc2-4bac-bcee-043d7fb68bd9" />

В журнале печати были зафиксированы множественные задания от пользователя Guest. Задания появлялись массово и приводили к бесконечной печати листов с некорректным содержимым/мусором. Наличие пользователя Guest в данном случае рассматривалось как признак нештатной или прямой сетевой печати, а не как штатная активность пользователя.

## Export log for detailed analysis

Для проведения дополнительной проверки был выполнен экспорт журнала заданий.

Журнал был экспортирован для проведения дополнительного анализа.
<img width="764" height="205" alt="Без имени" src="https://github.com/user-attachments/assets/d0cea0a4-8bc8-4bcf-948d-c331bc954a7a" />

Анализ экспортированных данных показал:

* все видимые записи идут от Guest;
* один и тот же документ повторяется много раз;
* многократное повторение данного задания соответствовало наблюдаемому инциденту с массовой печатью некорректного содержимого;

Полученные данные подтвердили факт многократной отправки одного и того же задания печати от пользователя Guest, однако журнал устройства не содержал достаточной информации для идентификации источника. В связи с этим была выполнена проверка сетевых протоколов и способов подключения к принтеру.

## Security Event Log Analysis

Для уточнения источника появления заданий печати был выполнен экспорт журнала событий безопасности принтера (Security Event Log).

Файл журнала был сохранён в текстовом формате и открыт для анализа.
<img width="810" height="185" alt="image" src="https://github.com/user-attachments/assets/d020649c-a73e-4fba-9f6b-9137013cb564" />

В журнале были обнаружены многочисленные записи, содержащие следующие параметры:
```text
user="Guest"
source_IP="0.0.0.0"
```
Указанные значения встречались в большинстве записей, относящихся ко времени возникновения инцидента.

Анализ журнала показал:

* присутствие событий, связанных с учётной записью Guest;
* наличие записей, в которых источник задания не был определён устройством (source_IP = 0.0.0.0);
* по данным журнала определить устройство-источник не удалось;
* в проанализированных записях журнала не были обнаружены признаки изменения конфигурации устройства или неудачных попыток доступа к веб-интерфейсу;

Для ускорения анализа был разработан скрипт `hp_joblog_parser.py`, который позволяет автоматически выявлять:

* задания от Guest;
* повторяющиеся задания;
* наиболее активных пользователей;
* наиболее активные IP-адреса;
* общее количество заданий печати.

## Checking the device configuration

После анализа журнала была выполнена проверка конфигурации сетевых сервисов принтера.
<img width="1183" height="465" alt="image" src="https://github.com/user-attachments/assets/51e3079b-6852-4c75-a3a2-75e95894c39a" />

В ходе проверки было установлено, что на устройстве активированы следующие сервисы:

* RAW/JetDirect (TCP/9100);
* LPD;
* IPP;
* IPPS;
* AirPrint;
* Web Services Print.

Особое внимание было уделено протоколу RAW/JetDirect (TCP/9100), поскольку он позволяет отправлять задания напрямую на устройство без использования централизованного сервера печати. 
При использовании данного протокола принтер может регистрировать задания без передачи информации о реальном пользователе или рабочей станции, что потенциально объясняет появление записей от пользователя Guest и отсутствие данных об источнике задания.

---

## Conclusion

Анализ показал, что устройство поддерживает несколько протоколов сетевой печати, включая RAW/JetDirect (TCP/9100), позволяющий передавать задания напрямую на принтер.

Учитывая наличие многочисленных записей от пользователя Guest и отсутствие информации об источнике заданий, одной из наиболее вероятных причин инцидента является использование прямой сетевой печати без передачи данных аутентификации пользователя.

Для упрощения дальнейшего анализа журналов HP был разработан скрипт (`hp_joblog_parser.py`).
